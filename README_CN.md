欢迎来到Data2Restful项目的原始仓库。Data2Restful是一个旨在将数据服务发布、管理、维护、测试等多项业务集成于一体的数据服务管理系统。将数据服务发布平台化是为了简化用户发布数据服务的流程，而使用者不必关注于业务逻辑之外的内容。本系统能够以更简洁的方式帮助用户快速高效地发布和管理数据服务，从而提高用户的使用体验。该项目目前仍处于开发的早期阶段，尚未准备好投入生产使用。

## 什么是Data2Restful?

Data2Restful是一个数据服务管理系统，可以简化使用者对数据库的操作流程，具有以下功能：

- **异构数据源的统一设计窗口**：在Data2Restful系统中，用户在添加不同来源的数据源后，可以根据不同数据源的特点，在相同的窗口中编写各自的操作代码。
- **数据服务接口的发布、管理、维护和测试**：Data2Restful为所有的数据服务提供了贯穿其整个生命周期的完整流程。数据服务接口一般是一个对数据源进行增删改查的具体操作。Data2Restful系统提供了一个特殊的方式，将这一数据服务接口打包发布至k8s集群中，并以Restful的方式提供了一个api。其他用户可以通过这个api调用发布者设计的数据服务。此外，Data2Restful还提供了一个数据服务的测试平台。用户可以在发布数据服务后，在测试平台中更简便地对当前数据服务进行测试，以验证其正确性。
- **数据服务功能扩展**：Data2Restful系统为数据服务分组提供了一个有趣的插件机制。用户能够以一种极其简便的方式，为数据服务添加扩展功能的插件。目前我们设计了四种类型的插件：
  1. key-auth：这一插件为数据服务中的接口添加了关键字验证的要求。即是，访问该数据服务的接口需要在请求头中带有该关键字以及相应的凭据。
  2. acl：这一插件为数据服务提供了访问权限控制的功能，主要是针对与黑白名单的过滤。使用该功能需要基于完整的用户模块，本项目中提供了简单的示例。
  3.  rate-limit：这一插件为数据服务提供了访问频率控制的功能。启用访问速率控制插件时，Data2Restful将在网关层面限制对当前数据服务的访问频率，以提高数据服务的安全性。
  4. dynamic-expansion：这一插件为数据服务提供了水平动态扩展的功能。启用动态扩展插件时，数据服务会根据现有节点的资源占用情况动态调整节点数量，以保证当前数据服务的高可用性。
